NG-SPORTS是什么?

四锅头同学闲暇之余制作的的Angular5 + springboot运动健身网站

本网站用于学习,巩固该同学ng2+的知识,也是督促该同学减肥的重点项目

NG-SPORTS都用了什么东西？

前端方面: Angular5 + SCSS + NZ-ZORRO + HighCharts

服务端方面: springboot + spring + mybatis + mysql

跨源解决策略: CORS

开发工具: vs code + eclipse; 构建和部署工具：angular/cli

服务器: 阿里云centos 网站IP: 47.97.166.254

NG-SPORTS模块如何划分?

根模块: appModule 组件包括 header(头部导航) home(主页) author-introduce(作者简介)

懒加载子模块: loginModule(登录模块) registerModule(注册模块) planModule(会员减肥计划) healthyModule(健身动作) healthyModule组件包括 add-comment(评论) healthy(动作大全) picture(动作卡片 复用于home组件中)

NG-SPORTS中的一些知识

本项目运用了Angular中的service、pipe、router、FormsModule、ReactiveFormsModule、以及部分生命周期钩子的知识

数据传递与存储:

1.组建交互中运用了路由传参、@input、@output、service传参;

2.前后端数据传递运用了HttpClient的get 和 post方法 采用CORS处理跨域;

3.客户端缓存利用了local Storage存储信息;

UI采用了阿里云 蚂蚁风格的 NG-ZORRO 0.6.14版本;图表为HighCharts;

布局上面大方向采用了Float布局(NG-ZORRO栅格),训练效果模块采用Flex布局(自练习)

NG-SPORTS中遇到的困难和问题

1.感觉NG-ZORRO还不够强大,很多细节不够完善，需要自己修改样式(:host ::ng-deep )

2.由于不想使用JSONP,也不想麻烦搭建nginx,最后采取H5 CORS方式，在服务端设置添加请求头完成跨域访问

3.HighCharts如放在oninit钩子中启动会先于nz-row使显示位置错误，需设置timeout使其异步延后执行

4.npm与cnpm混用 Angular/CLI容易出现问题,尽量改用npm进行下载(必要时需科学上网)

IPORT是什么?

i-port是日本野村证券的一款证券分析类超大型网站,原项目为Flex项目,本次开发属于技术重构.

重构页面700+,六元素科技负责前端页面与逻辑重构

IPORT都用了什么东西？

框架要素: AngularJS1.5 + SCSS + HTML5 + TypeScript2.0

重要插件: Highcharts、SpreadJS、wijmo、部分uib-bootstrap(uib-tooltip、uib-modal等)

CSS: 自作css 布局为基于table-cell 24等分的表格式布局

构建工具: gulp

webService端: C# C混合开发

我在IPORT负责了什么?

2017.03-2017.06(量产MOCK画面阶段).负责画面编写、测试

2017.07-2017.10(MOCK画面大量生产阶段) 负责小部分复杂画面的编写,对部分同事的问题进行帮助和技术支持

2012.11-2018.02(画面正式开发阶段) 进入共通函数小组进行共通函数的编写，包括普通function、angularjs directive、项目共通service以及部分插件的二次封装等

IPORT有什么难点和技术难题?

1.spreadjs 由于是收费软件,网络上的呼声不高，例如spreadjs行选择问题网络没有答案，需要paint重新手绘,则会带来新的卡顿问题，用户体验不好。总结:spreadjs坑太多

2.AngularJS优化问题,无论新老ng,这都是一个令人头疼的问题，新版本模块化更清新还好解决,在IPORT中则只能通话少调用directive fliter尽量不使用差值表达式这一系列规则,可效果并不显著

Smart Home是什么?

Smart Home是为日本若松连锁酒店提供的一套智能居住方案(ipad控制酒店房间) + 酒店后台ERP系统 的完整酒店自动化方案,目前已在日本知多、函館等多地上线

Smart Home都用了什么东西？

顾客(ipad): H5 + CSS3 + ionic1 + gulp

后台ERP: Java快速开发平台 经典组合JSP + spring mvc + spring + mybaits + shiro + Ehcatch + mysql

我在Smart Home 负责什么？

ipad端: 部分画面的编写 + 部分与后端api接口对接

后台:

1. 首页房间安排的全部工作.

2. 酒店餐饮、库管系统的全部工作.

3. 旅游订单模块的全部工作.

4. 酒店所有报表(10余种Excel)导出.

Smart Home有什么难点和技术难题?

1.ERP系统首页的拖拽分房、上方顾客信息滚动条的实现

2.餐饮订单系统和顾客住宿中产生的各种复杂的税的计算

3.Excel导出功能的制作,其中包括了各种酒店的复杂数据处理.报表导出插件 JXLS
